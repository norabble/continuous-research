> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship the read-only per-instance site (spec:
[`../specs/2026-07-06-live-site-design.md`](../specs/2026-07-06-live-site-design.md)):
engine `site` subcommand + pure renderer + scaffolded Pages workflow —
**deployed on `continuous-research-sample` for prototype review as soon as
the index page is usable (end of Task 5)**, before detail pages and
scaffold/docs work.

**Architecture:** `site` gathers `SiteData` through `GitHubPort` (one new
read: `listOpenPullRequests`), a pure `renderSite(SiteData): SiteFile[]`
emits static HTML+CSS into `_site/`, and a thin scaffolded `site.yml`
deploys via `actions/deploy-pages`. Markdown is treated as untrusted
(agent-written): raw HTML pre-escaped, claim annotations stripped, unsafe
link schemes neutralized.

**Tech Stack:** TypeScript ESM strict · Node ≥ 22 · vitest · `marked`
(new runtime dep, zero transitive) · Octokit (existing) · GitHub Pages.

## Global Constraints

- ESM only; `import type` for type-only imports (`verbatimModuleSyntax`).
- Ports-and-adapters: renderer and gathering logic pure; I/O only in
  `cli.ts` / `github.ts` (phase-1 design rule 1).
- CONCEPT vocabulary in code/comments; **site-facing copy uses the spec's
  translation table** (proposed update / awaiting the author's review /
  what this changes / evidence record / edition / maintenance).
- Trust rule (spec): only `[bot]`-authored PRs appear on the site at all.
- Fail closed: any gather/render error exits 1; deploy keeps last good site.
- `npm run check` green before every commit; commits end with
  `Co-Authored-By: <the model you are running as> <noreply@anthropic.com>`.
- Prototype release = tag `v0.1.4` (Task 5); final = `v0.1.5` (Task 8).
- After any `src/scaffold.ts` change, run the `scaffold-template-reviewer`

```ts
import { Marked } from "marked";

const SAFE_HREF = /^(https?:|mailto:|#|\.{0,2}\/|[a-z0-9_-]+(\/|\.|$))/i;
const ANNOTATION_LINE = /^\s*<!--\s*claim:[\s\S]*?-->\s*$/gm;

const escapeHtml = (s: string): string =>
  s.replaceAll("&", "&amp;").replaceAll("<", "&lt;").replaceAll(">", "&gt;");

export function renderUntrustedMarkdown(md: string): string {
  const source = escapeHtml(md.replace(ANNOTATION_LINE, ""));
  const marked = new Marked({
    async: false,
    walkTokens: (token) => {
      if ((token.type === "link" || token.type === "image") && "href" in token) {
        // Escaping above protects attributes; this guards the URL *scheme*.
        if (!SAFE_HREF.test(String(token.href))) token.href = "#";
      }
    },
  });
  return marked.parse(source) as string;
}
```

Note: pre-escaping means markdown *autolinks* of escaped text still work
and raw HTML degrades to visible text — exactly the "passthrough disabled"
stance. Verify against the installed marked version's token shape (v12+:
`token.href` on link/image tokens); adjust the guard, keep the tests.
